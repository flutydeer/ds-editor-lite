# QWidget 覆盖层就地编辑分阶段实施计划

## 实施原则

每个阶段都是独立、可构建、可手工验收的纵向切片，并在 Debug 构建通过后单独提交。旧实现仅在对应入口完成迁移后删除，避免一次改动覆盖全部场景。

## 第一阶段：通用覆盖层与轨道名称 ✅ 已完成

- ✅ 新增 `InlineTextEditOverlay`：普通子 `QWidget` 覆盖层，包含专用 `QLineEdit`，不使用任何顶层窗口标志。
- ✅ 新增 `InlineEditLabel`：使用内部 `QLabel` 显示文本，双击后创建覆盖层；提供文本、字体、对齐、validator、编辑角色和提交信号。
- ✅ 完成 Enter 提交、Esc 取消、FocusOut 提交、无变化不提交、输入法兼容和重复结束保护（`m_submitted`）。
- ✅ 使用项目 `Menu` 提供剪切、复制、粘贴、撤销；菜单打开期间不因 `FocusOut` 提交，关闭后恢复编辑焦点。
- ✅ 仅将轨道名称从旧 `EditLabel` 迁移到新控件。覆盖层定位到轨道列表 viewport。
- ✅ 轨道列表滚动、纵向缩放、重排、尺寸变化立即提交并退出（`finishTrackNameEditing` + 信号连接）。
- ✅ 轨道名允许为空；一次有效修改只产生一次属性 action。
- M/S 按钮固定 24×24，标签使用 `stretch=1` 占满剩余空间。
- 左外边距 `12→4`，与标签内部 8px 边距合计保持 12px 文本偏移。
- QSS 选择器追加 `InlineEditLabel` 支持。

验收点：

- ✅ 主窗口始终保持活动状态。
- ✅ 右键菜单能够粘贴，关闭菜单后仍可继续编辑。
- ✅ 滚动、缩放、点击其他位置立即提交。
- ✅ Esc 不修改名称，Enter 和焦点移出各只提交一次。
- ✅ 轨道重排、删除和窄模式切换无错位或崩溃。

阶段完成后运行 Debug preset 完整构建并通过。提交 `cb439599`。

## 第二阶段：底部面板剪辑名称 ✅ 已完成

- ✅ 将剪辑名称从常驻 `LineEdit` 改为 `InlineEditLabel`，复用第一阶段覆盖层，并保留原 128px 固定宽度和 Tooltip。
- ✅ 覆盖层挂到剪辑工具栏本地宿主，底部面板嵌入或独立时均保持正确坐标与 QSS。
- ✅ 编辑会话绑定开始编辑时的 clip id；切换活动剪辑和外部属性刷新前先提交，禁止写入错误剪辑。
- ✅ 面板拆分、重新吸附、窗口/面板尺寸变化、滚轮和窗口失活时立即提交并退出。
- ✅ 剪辑名允许为空；名称无变化或目标已删除时不调用属性 controller。
- ✅ `editRole/groupPos` 传递给覆盖输入框，编辑态背景、边框和圆角与 ControlGroup 一致。

验收点：

- 嵌入底部面板和独立窗口状态下均可双击编辑。
- 右键复制、粘贴和撤销正常。
- 编辑过程中切换剪辑，结果写入原剪辑且不污染新剪辑。
- 拆分、吸附、调整窗口大小时立即提交，无覆盖层残留。
- 未修改内容时不产生撤销记录。

阶段完成后 Debug 构建通过，独立提交。

## 第三阶段：钢琴卷帘歌词 ✅ 已完成

- ✅ `PianoRollGraphicsView` 持有唯一 viewport 覆盖编辑器和稳定的目标 note id。
- ✅ `NoteView` 移除歌词专用 `QGraphicsProxyWidget/QLineEdit`，只保留编辑绘制状态。
- ✅ 使用 `mapFromScene(sceneBoundingRect())` 转换像素坐标，设置 40×20 最小编辑尺寸并约束在 viewport 内。
- ✅ 编辑期间停止绘制底层歌词，覆盖输入框通过 `Lyric` 角色应用独立 QSS。
- ✅ 歌词提交前执行 `trimmed()`；清空时使用音符自身语言对应的默认歌词。
- ✅ Tab/Shift+Tab 按音符起始位置和 id 稳定导航到下一/上一音符。
- ✅ 滚动缩放、viewport 尺寸、选择、工具、音符属性和剪辑上下文变化立即提交。
- ✅ 使用固定的覆盖层信号连接，删除 NoteView 临时连接和重复提交路径。

验收点：

- 中文输入法候选确认不会提前结束编辑。
- 右键菜单可纯鼠标粘贴。
- 横纵滚动、横纵缩放立即提交，不跟随重定位。
- Tab 和 Shift+Tab 连续录入歌词正常。
- 空歌词恢复对应语言默认歌词。
- 连续编辑同一音符不会产生重复 action。

阶段完成后 Debug 构建通过，独立提交。

- ✅ 未编辑发音以 original 为显示基准，原样退出不会误创建人工发音覆盖。

验收点：

- 歌词和发音之间切换时前一个字段正确提交。
- 发音右键粘贴正常，菜单关闭后继续编辑。
- 清空人工发音后恢复原始发音及对应显示颜色。
- 滚动、缩放、删除音符和切换剪辑时没有残留编辑器。
- 重复进入和退出不会多次修改模型。

阶段完成后 Debug 构建通过，独立提交。

## 第五阶段：剩余 EditLabel 数值入口

- 将速度、拍号、播放位置、声像和增益从旧 `EditLabel/EditDialog` 迁移到 `InlineEditLabel`。
- 每类字段使用独立验证规则，不再使用通用且生命周期错误的栈上 `QDoubleValidator`。
- 速度仅接受业务允许范围内的数字；拍号验证正整数 `分子/分母`；播放位置验证三段时间格式；声像和增益保留现有解析、截断和规范化显示。
- 非法或不完整输入在外部事件发生时退出编辑并恢复当前模型值，不转换为 `0`，不发送业务修改。
- 标题栏播放、窗口尺寸变化、混音台滚动或相关控件交互均立即提交当前合法输入。
- 为各 `editRole` 完善 QSS，保持音乐字体、对齐、边框、选择色和上下文菜单样式。

验收点：

- 五类字段均保持原有合法输入和规范化显示行为。
- 非法输入不改变模型。
- 标题栏和混音台右键粘贴正常。
- 播放或操作其他控件时编辑立即结束。
- 主窗口标题栏不会因编辑器出现而失去活动状态。

阶段完成后完整构建并暂停，等待用户确认。

## 第六阶段：清理与最终回归

- 确认没有调用方后删除 UI Controls 中旧的 `EditLabel/EditDialog` popup 实现；不删除 FillLyric 模块自己的同名控件。
- 清理旧 proxy 成员、临时信号连接、无效 include、TODO 和不再使用的 QSS 选择器。
- 检查所有有效编辑只产生一次原有业务 action，撤销/重做结果正确。
- 使用项目标准 Debug preset 完整构建。
- 按用户选择不新增自动化测试目标，执行全部入口的交叉手工回归。

## 固定行为约定

- 编辑器内部文本操作、输入法和自身右键菜单不属于“外部事件”。
- 滚动、缩放、窗口或面板尺寸变化、选择/工具/数据上下文变化、目标移动或删除、窗口失活都属于外部事件，立即提交并退出。
- 普通重绘和 QSS 刷新不触发提交。
- 目标已失效且无法提交时安全取消。
- 轨道名和剪辑名允许为空；空歌词恢复语言默认歌词；空发音恢复原始发音。
